PLATAFORMA CORPORATIVA DE CALIDAD
RESUMEN DE AVANCE DEL SISTEMA
Fecha de corte: 2026-03-22

1. OBJETIVO GENERAL

El sistema fue disenado para gestionar anomalias, acciones, tareas, notificaciones y trazabilidad operativa dentro de un entorno corporativo. La arquitectura actual ya permite operar localmente en planta y, al mismo tiempo, deja preparada la migracion futura a infraestructura cloud sin cambios fuertes de arquitectura.

2. ARQUITECTURA GENERAL

Backend:
- Django
- Django REST Framework
- PostgreSQL
- JWT para autenticacion

Frontend:
- React
- TypeScript
- Vite
- PWA base

Despliegue:
- Docker Compose local
- Nginx para servir frontend y hacer proxy de API
- Gunicorn para backend
- Volumenes persistentes para datos, adjuntos, logs y backups

4. WORKFLOW DE ANOMALIAS SOPORTADO

Etapas:
1. Registro
2. Contencion
3. Verificacion inicial
4. Clasificacion
5. Analisis de causa
6. Propuestas
7. Plan de accion
8. Ejecucion y seguimiento
9. Resultados
10. Verificacion de eficacia
11. Cierre
12. Estandarizacion y aprendizaje

Estados principales:
- registered
- in_evaluation
- in_analysis
- in_treatment
- pending_verification
- closed
- cancelled
- reopened

Reglas ya implementadas:
- Toda anomalia tiene ID unico
- Toda transicion relevante se audita
- No se puede cerrar sin verificacion minima
- No se puede completar aprendizaje sin registrar learning
- Si falta informacion obligatoria, el backend bloquea la transicion
- La logica critica del workflow vive en backend

7. DESPLIEGUE LOCAL CLOUD-READY

Separacion ya definida:
- aplicacion backend
- frontend
- base de datos
- archivos adjuntos
- configuracion por variables de entorno
- logs
- backups

Buenas practicas ya aplicadas:
- no hardcodear IPs en configuracion principal
- variables de entorno centralizadas
- rutas portables
- media y static fuera del codigo
- logs persistentes
- backups con carpeta dedicada
- misma origin para frontend y backend en despliegue productivo local
